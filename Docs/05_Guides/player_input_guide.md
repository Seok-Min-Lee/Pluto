# 🎮 플레이어 조작 시스템 가이드 (Player Input Guide)

이 문서는 플레이어 조작 방식과 액션 피드백 정책을 정의합니다.

---

## 1. 기본 조작 체계 (Keyboard & Mouse)

| 액션 | 입력 (Default) | 설명 | 비고 |
| :--- | :--- | :--- | :--- |
| **이동 (Move)** | W, A, S, D | 캐릭터 8방향 이동. | |
| **공격 (Attack)** | Mouse Left | 기본 공격 (Attack A, B, C). | |
| **특수 공격 (Special)**| Mouse Right | Attack Special 발동. | |
| **마법 (Magic)** | Q Key | Magic 기술 발동. | |
| **돌진 (Dash)** | Space Bar | 지정된 방향으로 돌진. | 무적 프레임 존재 |
| **상호작용 (Interact)**| E Key | NPC 대화, 문 열기 등. | |

---

## 2. 조작 매커니즘 (Mechanics)

### A. 입력 선입력 (Input Buffering)
- 액션 후딜레이 중 입력된 다음 명령을 기억하여 동작 가능 시점에 즉시 실행합니다.
- 적용: 공격 연계, 공격-대시 연계 등.

### B. 대시 시스템 (Dash System)
- **시간 동기화**: 대시 이동과 애니메이션이 정밀하게 일치하도록 제어됩니다.
- **경직 시스템 (End-lag)**: 대시 종료 직후 짧은 경직을 주어 동작의 무게감을 제공합니다.
- **대시 캔슬 (Dash Canceling)**: 공격 중 대시 입력을 통해 동작을 즉시 중단하고 회피할 수 있습니다.

### C. 마법 운용 및 혈석 회수 (Magic & Bloodstone)
1. **비설치형 (Target-Embedded):**
   - **로직:** 적에게 투사체를 맞추면 혈석이 박힙니다.
   - **회수:** 효과 종료 또는 적 사망 시 혈석이 바닥에 떨어지며, 플레이어가 **직접 접근하여 회수**해야 다시 사용할 수 있습니다.
2. **설치형 (Deployable / Area):**
   - **로직:** 특정 위치에 수정탑을 소환하거나 가스 안개를 생성합니다.
   - **회수:** 효과가 지속되는 동안 해당 혈석 슬롯은 비활성화(점유) 처리되고 효과가 종료되면 혈석이 바닥에 떨어지며, 플레이어가 **직접 접근하여 회수**해야 다시 사용할 수 있습니다.

---

## 3. 핵심 시스템 요약

### A. 하이브리드 제어 및 복귀 캔슬 (RC)
스크립트 중심의 정밀 액션과 애니메이터의 시각적 유연성을 결합한 구조를 사용합니다. 공격 후반부에서 이동 입력 시 즉시 동작이 취소되며 이동으로 전환되는 '복귀 캔슬'이 적용됩니다.

### B. 조작 우선순위 및 잠금 (Locking)
대시 및 공격 중에는 플레이어의 의사가 없는 이동/회전 간섭을 차단하여 동작의 일관성을 유지합니다. 캐릭터는 이동 시 입력 방향을, 공격 시 커서 방향을 즉시 추종합니다.
